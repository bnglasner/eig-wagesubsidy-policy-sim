# EIG Document Review Process

Full reference for the cover sheet template, pre-draft planning, review chain, and final handoff checklist for EIG research publications.

---

## 1. Publication Cover Sheet Template

Every EIG research publication must have a completed cover sheet before entering the review process. Fill out all fields. Leave none blank — use "N/A" if a field does not apply.

| Field | Entry |
|-------|-------|
| **Document Title** | *(Working title of the publication)* |
| **Document Type** | *(Research report / Policy brief / Blog post / Fact sheet / Data explainer / Press release)* |
| **Primary Author(s)** | *(Names of lead researcher(s) writing the document)* |
| **Assisting Researcher(s)** | *(Names of others contributing analysis, data, or sections)* |
| **Project Lead** | *(EIG staff member responsible for overall project delivery)* |
| **Date Initiated** | *(Date the drafting process began, YYYY-MM-DD)* |
| **Target Release Date** | *(Planned external publication date, YYYY-MM-DD)* |
| **Target Audience** | *(Primary: e.g., policymakers, journalists, economists, general public)* |
| **Core Finding / Thesis** | *(1–2 sentence statement of the document's central argument or finding)* |
| **Key Data Sources** | *(List all major datasets used; include version/year)* |
| **Related EIG Projects** | *(List any prior EIG reports, data products, or ongoing projects this connects to)* |
| **External Partners** | *(Names of any co-authors, co-publishers, or external reviewers)* |
| **Embargo / Release Restrictions** | *(Any embargo date or distribution restrictions; if none, write "None")* |
| **Geographic Scope** | *(National / State / Metro / County / Other — specify)* |
| **Time Period** | *(Years or date range covered by the analysis)* |
| **Planned Outputs** | *(e.g., main report PDF, summary brief, blog post, data download, press release, social graphics)* |
| **Communications Lead** | *(EIG staff member coordinating external release, media, and social)* |
| **Legal / Compliance Review Needed?** | *(Yes / No — if yes, describe)* |
| **Current Status** | *(Pre-draft / In-draft / Internal review / External review / Final / Published)* |
| **Notes** | *(Any additional context, risks, or dependencies)* |

## 2. Pre-Draft Questions Checklist

Before beginning to write, confirm answers to these questions:

### Audience & Purpose
- [ ] Who is the primary audience for this document?
- [ ] What is the single most important thing they should take away?
- [ ] What action, if any, do we want readers to take after reading?
- [ ] Is there a specific news hook, policy moment, or data release this is timed to?

### Scope & Argument
- [ ] What is the central thesis or finding in one sentence?
- [ ] What are the two or three supporting points that make the case?
- [ ] What is the strongest counterargument, and how does the document address it?
- [ ] Is the scope (geography, time period, population) clearly defined and defensible?

### Data & Methods
- [ ] Are all data sources identified and accessible?
- [ ] Are the methods documented and replicable?
- [ ] Are there any data limitations that must be disclosed in the text?
- [ ] Has the data and code been saved in the project folder per EIG file management standards?

### Review & Timeline
- [ ] What is the planned release date, and is the timeline achievable?
- [ ] Who needs to approve this before it is released?
- [ ] Does this document require legal or compliance review?
- [ ] Are there external partners who need a pre-release copy?

## 3. Full Review Chain

EIG documents pass through the following stages in order. Each handoff requires the previous reviewer to sign off before the next stage begins.

```
[Author(s)]
    │  Draft complete; cover sheet attached
    ▼
[Project Lead]
    │  Reviews for argument, evidence, scope, and framing
    │  Returns edits → Author revises
    ▼
[Research Team Review]
    │  Peer review for analytical accuracy, data accuracy, methods
    │  Returns comments → Author revises
    ▼
[EIG Writing/Style Review]
    │  Checks all writing style, grammar, punctuation, citation format
    │  Returns line edits → Author accepts/revises
    ▼
[Communications Review]
    │  Reviews headline, lede, framing for audience; flags any reputational risks
    │  Coordinates planned assets (social graphics, press release, blog post)
    ▼
[Senior Leadership Sign-Off]
    │  Final approval of substance and release timing
    │  Must approve before any external sharing
    ▼
[Production / Design]
    │  Layout, figures formatted to EIG brand standards
    │  Final PDF generated; web version formatted
    ▼
[Final Proofread]
    │  One final read of the designed document for typos, figure numbering, broken links
    ▼
[Release]
```

### Reviewer Responsibilities

| Reviewer | Checks |
|----------|--------|
| Project Lead | Argument structure; evidence quality; scope; covers all pre-draft questions |
| Research Team | Analytical accuracy; correct use of data; methods defensibility; statistics |
| Writing/Style | EIG style compliance; grammar; punctuation; citation format; figure labels |
| Communications | Headline and lede clarity; audience framing; media angles; asset coordination |
| Senior Leadership | Strategic fit; reputational considerations; release timing; external partnerships |
| Production/Design | Brand standards; figure formatting; PDF quality; web layout |
| Final Proofread | Typos; figure/table numbering sequence; hyperlinks; page numbers |

## 4. Press Release Process

Press releases follow a condensed review chain and require coordinated timing with the main publication.

1. **Draft:** Communications team drafts release in parallel with final research review.
2. **Research accuracy check:** Primary author confirms all statistics, quotes, and findings in the release are accurate and match the final document.
3. **Leadership approval:** Senior leadership approves the release text and any attributed quotes.
4. **Distribution list:** Communications confirms wire service, media contacts, and partner notification list.
5. **Embargo management:** If under embargo, confirm all recipients have signed embargo agreement before sharing.
6. **Release timing:** Coordinate release of press release, publication PDF, web post, and social media simultaneously unless otherwise planned.

---

## 5. Final Handoff Checklist

Before handing any document to production or marking it as ready for release, verify:

### Content
- [ ] Cover sheet fully completed and attached
- [ ] All pre-draft questions addressed in the text
- [ ] Central thesis clearly stated in the introduction
- [ ] All data sources cited; all figures have source lines
- [ ] Limitations and caveats disclosed where appropriate
- [ ] All reviewer comments resolved or explicitly noted as deferred

### Style
- [ ] EIG Writing Style rules applied (see eig-writing-style.md)
- [ ] All citations formatted per EIG citation style (see eig-citation-style.md)
- [ ] Figure labels: "Figure N." prefix; sentence-case captions; source lines
- [ ] No contractions; no elegant variation; active voice throughout

### Logistics
- [ ] All planned companion assets identified (press release, blog post, social graphics, data download)
- [ ] Release date confirmed with communications and leadership
- [ ] External partners and co-publishers notified per agreement
- [ ] Embargo instructions communicated to all pre-release recipients
- [ ] Final PDF saved to project folder with version date in filename (YYYY-MM-DD_ProjectName_Final.pdf)
- [ ] Data and code archived in project folder per EIG data management standards
