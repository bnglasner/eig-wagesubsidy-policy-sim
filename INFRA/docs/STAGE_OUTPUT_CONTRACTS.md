# Stage Output Contracts

This file is the source of truth for required stage inputs/outputs and minimum validation checks.
`INFRA/scripts/validate_stage_contracts.sh` must remain synchronized with the artifact patterns listed below.

Stages are activated by the `Project scope tier` set in `INFRA/docs/PIPELINE_LANGUAGE_PROFILE.md`:
- **Tier 1** (Descriptive/Blog): stages 01, 02, 05 required
- **Tier 2** (Analytical Brief): stages 01, 02, 03, 05 required
- **Tier 3** (Full Research Paper): all stages required

## Stage 01: Data Preparation *(All Tiers)*

| Stage | Required Inputs | Required Outputs | Minimum Validation | Fail Condition |
|------|------------------|------------------|--------------------|----------------|
| `01_data_preparation` | Source files listed in `INFRA/docs/DATA_SOURCES_OUTPUTS_CATALOG.md` | `WORKSPACE/data/processed/analysis_dataset.*`, `WORKSPACE/output/data/intermediate_results/data_dictionary.*`, and `INFRA/quality_reports/data_report.html` | Dataset loads successfully; required key identifiers are present and non-null; data report exists and is non-empty | Missing dataset, data dictionary, or data report |

## Stage 02: Descriptive Analysis *(All Tiers)*

| Stage | Required Inputs | Required Outputs | Minimum Validation | Fail Condition |
|------|------------------|------------------|--------------------|----------------|
| `02_descriptive_analysis` | `WORKSPACE/data/processed/analysis_dataset.*` | `WORKSPACE/output/tables/main/descriptive_summary.*` and `WORKSPACE/output/data/intermediate_results/descriptive_qc.*` | Row counts and summary stats generated without errors | Missing descriptive summary or QC artifact |

## Stage 03: Main Estimation *(Tier 2+)*

| Stage | Required Inputs | Required Outputs | Minimum Validation | Fail Condition |
|------|------------------|------------------|--------------------|----------------|
| `03_main_estimation` | `WORKSPACE/data/processed/analysis_dataset.*` | `WORKSPACE/output/tables/main/main_estimates.*` and `WORKSPACE/output/data/intermediate_results/model_diagnostics.*` | Main model converges/runs; core claim coefficients are produced | Missing estimate table or model diagnostics |

## Stage 04: Robustness And Heterogeneity *(Tier 3 only)*

| Stage | Required Inputs | Required Outputs | Minimum Validation | Fail Condition |
|------|------------------|------------------|--------------------|----------------|
| `04_robustness_heterogeneity` | Stage `03` model inputs/specification artifacts | `WORKSPACE/output/tables/appendix/robustness_results.*` and `WORKSPACE/output/tables/appendix/heterogeneity_results.*` | Alternate specs run and output tables generated | Missing robustness or heterogeneity table |

## Stage 05: Figures And Tables *(All Tiers — main outputs; appendix outputs Tier 3 only)*

| Stage | Required Inputs | Required Outputs | Minimum Validation | Fail Condition |
|------|------------------|------------------|--------------------|----------------|
| `05_figures_tables` | Stage `02`-`04` output artifacts | `WORKSPACE/output/figures/main/*`, `WORKSPACE/output/figures/appendix/*`, `WORKSPACE/output/tables/main/*`, `WORKSPACE/output/tables/appendix/*`, and `INFRA/quality_reports/output_report.html` | Required figure/table files exist and are non-empty; output report exists and is non-empty | Missing required final figure/table artifacts or output report |

## Notes

- File suffixes are project-specific and may be `.csv`, `.parquet`, `.rds`, `.md`, `.png`, etc.
- If a project needs different artifact names, update this file and stage READMEs together.
- Use `INFRA/scripts/validate_stage_contracts.sh` to check artifact presence against this contract.
- Template default is warn mode: `STAGE_CONTRACT_MODE=warn` (default behavior).
- After project scripts are implemented, switch to strict enforcement: `STAGE_CONTRACT_MODE=fail`.
- The validator reads `Project scope tier` from `INFRA/docs/PIPELINE_LANGUAGE_PROFILE.md` and only enforces contracts for active stages. Override with env var `EIG_PROJECT_TIER=1|2|3`.

## HTML Report Generation

The two HTML reports required by this contract are not auto-generated by the pipeline scripts.
Produce them from the templates in `INFRA/templates/`:

| Required Artifact | Source Template | When to Produce |
|---|---|---|
| `INFRA/quality_reports/data_report.html` | `INFRA/templates/data_report.html` | End of Stage 01 — fill with dataset summary, row counts, QC flags |
| `INFRA/quality_reports/output_report.html` | `INFRA/templates/output_report.html` | End of Stage 05 — fill with figure/table inventory and key findings |

Copy the template file to the quality_reports location and fill in the project-specific fields.
See the Stage 01 and Stage 05 READMEs for detailed instructions.
